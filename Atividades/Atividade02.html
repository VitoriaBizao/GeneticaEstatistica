Exercise 4

Compare and contrast coord_cartesian() vs. xlim() or ylim() when zooming in on a histogram. What happens if you leave binwidth unset? What happens if you try and zoom so only half a bar shows?

Code 
# Binwidth unset
p1 <- ggplot(df, aes(x = y)) + 
  geom_histogram(color = "black",
                 fill = "#cb73fa") +
  #coord_cartesian(ylim = c(0, 30000)) +
  labs(title = "Unset",
       tag = "A") +
  theme_minimal() +
  theme(axis.title = element_blank(),
        title = element_text(size = 8))

# Setting binwidth
p2 <- ggplot(df, aes(x = y)) + 
  geom_histogram(color = "black",
                 fill = "#cb73fa",
                 binwidth = .5) +
  coord_cartesian(ylim = c(0, 30000)) +
  labs(title = "binwidth = 0.5",
       tag = "B") +
  theme_minimal() +
  theme(axis.title = element_blank(),
        title = element_text(size = 8))

# Setting binwidth
p3 <- ggplot(df, aes(x = y)) + 
  geom_histogram(color = "black",
                 fill = "#cb73fa",
                 binwidth = 0.5) +
  coord_cartesian(ylim = c(0, 2.5),
                  xlim = c(10, 40)) +
  labs(title = "bin, y and xlim set",
       tag = "C") +
  theme_minimal() +
  theme(axis.title = element_blank(),
        title = element_text(size = 8))

# Binwidth unset
p4 <- ggplot(df, aes(x = y)) + 
  geom_histogram(color = "black",
                 fill = "#cb73fa") +
  coord_cartesian(xlim = c(0, 15)) +
  labs(title = "xlim = c(0, 15)",
       tag = "D") +
  theme_minimal() +
  theme(axis.title = element_blank(),
        title = element_text(size = 8))

grid.arrange(p1, p2, p3, p4,
             ncol = 2,
             top = grid::textGrob("Testing coord_cartesian(), xlim() and ylim()",
                                   gp = grid::gpar(fontsize = 12, fontface = "bold")),
             left = grid::textGrob("Count",
                                   rot = 90,
                                   gp = grid::gpar(fontsize = 10, fontface = "bold")),
             bottom = grid::textGrob("Width (in mm)",
                                   gp = grid::gpar(fontsize = 10, fontface = "bold")))

Os argumentos x_lim e y_lim do servem como um filtro, onde apenas os valores que dentro desse filtro específico são plotados. Por exemplo, no gráfico D, apenas os diamantes cuja largura estão entre 5 e 15 são plotados. Portanto, o coord_cartesian funciona como um zoom, que calcula todas as estatísticas e geometrias do gráfico para os limites que o usurário especificar.

Por default, o geom_histogram() tem bin = 30, portanto, se não for especificado ele ira dividir os dados em 30 blocos, e dependendo do quanto for o zoom, mais ou menos das barras poderão ser vistas.

Usando o binwidth pode ocorrer de a barra ser dividida, mas quando se usa xlim isso não ocorre pois os dados fora dos limites estabelecidos não são utilizados no plot.

As funções mean() e sum() não conseguem lidar com dados faltantes, e portanto não retornam o resultado, apenas um NA. Quando declaramos na.rm dizemos para que a função remova os NAs e faça a conta sem eles, e assim ela consegue fazer o cálculo.
